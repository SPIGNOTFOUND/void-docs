Getting Started

Ready to bring Void Bot into your server and experience its full potential? Follow these detailed steps to get started on the right foot.

Prerequisites

Before inviting Void Bot to your server, ensure you meet the following requirements:

  • You are the server owner or have administrator permissions. If you're not the main owner and the main owner is unavailable, you can still set up the bot by joining our support server where special whitelists can be created.
  • The bot's role is set as the highest in your server hierarchy. This ensures Void Bot has the necessary permissions to function effectively and protect your server.
  • You have a Discord account with access to the server where you want to add Void Bot.

Installation

Inviting Void Bot to your server is a breeze—easier than mixing sugar in water, we promise! Follow these simple steps:

  1. Visit the Void Bot invitation page. This is where you'll find everything you need to add the bot to your server.
  2. Click the "Add to Server" button. This will initiate the process of inviting Void Bot to your Discord server.
  3. Select your server from the dropdown menu. If you have multiple servers, make sure to pick the right one where you want Void Bot to reside.
  4. Grant the necessary permissions. Don't worry—we only request the permissions essential for Void Bot to function properly and keep your server safe.
  5. Click "Authorize" and watch the magic happen! Void Bot will join your server and begin setting things up for you.

Initial Setup

Once Void Bot is in your server, it's time to configure it to your specific needs. Follow these steps for a smooth initial setup:

  1. Run the command v-setup in your server. This command triggers an ephemeral guide that will walk you through the entire setup process, ensuring you don't miss any crucial steps.
  2. The bot will automatically create all necessary logging channels. These channels are where you'll find detailed records of all actions and events happening in your server, providing complete transparency and accountability.
  3. A quarantine role will be created by the bot. This role is used to isolate potentially problematic users or situations, helping to maintain server safety without disrupting the overall community experience.
  4. Void Bot will lock its (quarantine role) position right below its own role to ensure except Main owner nobody can mess up things. This ensures the bot has the necessary authority to execute its protective functions effectively.
  5. If you're on a special whitelisted server (due to the main owner being unavailable), the bot will ping the person who initiated the setup instead of the server owner for specific actions. This ensures continuity of management even in such cases.
  6. Follow the detailed guide provided by the bot to fine-tune your settings. We've added extra steps and explanations to make the setup process as smooth and user-friendly as possible.

Core Concepts

Void Bot's architecture is built around several core concepts that make it both powerful and flexible. The bot follows a modular design, allowing for easy updates and customization without affecting core functionality.

At its heart, Void Bot uses an event-driven system to respond to server activities in real-time. This ensures that actions like raids or rule violations are detected and addressed immediately. The bot's permission hierarchy system ensures that only authorized users can access specific commands, maintaining server security.

Another fundamental concept is the bot's data handling methodology. Void Bot employs encrypted storage for all server data and logs, ensuring privacy and security. The bot also implements rate-limiting to prevent abuse and ensure fair usage across all servers.

Finally, Void Bot's commitment to open-source principles means that anyone can suggest features and contribute to its development. The community plays a crucial role in shaping the bot's future by providing feedback and feature requests. This collaborative approach has been instrumental in making Void Bot what it is today.the bot code is not public itself.

Troubleshooting

Common issues and solutions when using Void Bot:

  • Bot isn't responding: Ensure the bot has the necessary permissions and that its role is positioned correctly in your server hierarchy.
  • Commands not working: Check if you have the required permissions for the command. Some commands are restricted to specific roles or users.
  • Logging not functioning: Verify that the logging channels created by the bot have the correct permissions and that the bot's role is positioned above other roles.
  • Persistent issues: Contact our support team through the official Discord server for assistance with more complex problems.

How does the global ban system work?

What permissions does Void Bot require?

Void Bot requires a set of permissions to function effectively. These include managing roles, reading message history, sending messages, and other standard moderation permissions. The bot will not request any permissions it doesn't need for its operation.

You can join our official Discord server and create a ticket in the appropriate channel. Our development team actively reviews community feedback and uses it to guide the bot's development roadmap.
